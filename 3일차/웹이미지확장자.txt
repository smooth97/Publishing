[웹에서 사용하는 이미지 확장자]

포토샵이나 일러스트에서 'Save For Web'을 이용하여 웹용 이미지로 저장하실 수 있습니다.
웹용 이미지는 무조건 RGB 컬러에 72ppi 해상도로 설정하셔야 합니다.

-JPG   : 웹에서 표준으로 사용되는 그래픽 파일의 확장자로, JPEG 압축 방식을 이용하여 압축한 이미지를 파일에 사용하는 파일 확장명입니다.
         Quality를 조절하여 용량을 확인하면서 저장이 가능합니다. 압축방식을 이용하는 이미지 저장 포맷이기 때문에 압축을 하면서 손실이 발생하게 되어
         화질이 떨어질 수 있습니다. 사진 등의 고해상도 이미지 저장에 주로 사용되며, 투명은 지원하지 않습니다. 만약 배경을 투명하게 만들어놓고 jpg로 저장하게 되면 배경이 흰색이 됩니다.

-GIF   : 비손실 압축 알고리즘을 사용하는 무손실 압축 포맷이기 때문에 기본적으로 이미지 저장시 원본 데이터와 달라지는 손실이 발생하지 않습니다.
         그러나 포맷 자체가 256색상 까지밖에 지원하지 않기 때문에 이미지 안에서 사용되는 컬러의 갯수가 256개를 넘어가는 이미지를 GIF 확장자로 저장할 시,
         컬러를 256색상으로 맞추고 줄이는 과정에서 손실이 발생해 화질이 떨어지는 것처럼 보일 수 있습니다. 투명은 표현이 가능하나 반투명은 불가하고,
         움직이는 애니메이션이 가능합니다. (ex : 움직이는 짤)

-PNG8  : GIF와 같이 비손실 압축 포맷입니다. 투명은 지원하지만 반투명을 불가하고, GIF와 같이 256색상까지 지원합니다.

-PNG24 : GIF와 같이 비손실 압축 포맷입니다. 투명, 반투명 모두 지원하며 트루컬러를 지원합니다. 그러나 압축을 하지 않기 때문에 파일 용량이 커질 수 있어 작은 아이콘 등에 많이 사용합니다.
         JPG와 같이 근래 가장 많이 사용되는 포맷이며 W3C에서 표준으로 지정한 포맷입니다. 웹용 이미지로 개발되었기 때문에 CMYK 컬러는 지원하지 않습니다.
         GIF의 특허권 때문에 개발된 포맷입니다. '핑'이라고도 읽습니다.


      ※ 이미지는 웹페이지 내에서 차지하는 용량이 매우 크기 때문에 이미지 포맷을 효율적으로 사용하지 않으면 웹페이지가 무거워질 수 있습니다.
         때문에 각 포맷의 장단점을 알고 접근한 뒤 사용하셔야 합니다.
